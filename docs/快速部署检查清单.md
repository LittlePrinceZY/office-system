## 🚀 部署步骤（按顺序）

### 第1步：推送代码到GitHub
- [ ] 在GitHub创建新仓库
- [ ] 本地执行 `git init`
- [ ] 本地执行 `git add .`
- [ ] 本地执行 `git commit`
- [ ] 本地执行 `git push`

### 第2步：部署前端到Vercel
- [ ] 登录Vercel
- [ ] 导入GitHub仓库
- [ ] 配置项目：
  - [ ] Framework: Vite
  - [ ] Build Command: `npm run build`
  - [ ] Output Directory: `apps/web/dist`
- [ ] 获取部署地址：`https://xxx.vercel.app`

### 第3步：部署数据库到Supabase
- [ ] 登录Supabase
- [ ] 创建新项目
- [ ] 获取数据库连接字符串
- [ ] 导入数据库Schema
- [ ] 开启Realtime功能

### 第4步：部署后端到Railway
- [ ] 登录Railway
- [ ] 创建新项目
- [ ] 导入GitHub仓库
- [ ] 配置：
  - [ ] Root Directory: `apps/api`
  - [ ] Build Command: `npm install && npm run build`
  - [ ] Start Command: `npm start`

### 第5步：配置环境变量

#### Railway环境变量：
```
DATABASE_URL=你的supabase连接字符串
JWT_SECRET=随机字符串
JWT_REFRESH_SECRET=随机字符串
PORT=3001
NODE_ENV=production
CORS_ORIGIN=https://你的前端地址.vercel.app
```

#### Vercel环境变量：
```
VITE_API_BASE_URL=https://你的后端地址.railway.app/api
VITE_WS_URL=https://你的后端地址.railway.app
```

### 第6步：测试系统
- [ ] 访问前端地址
- [ ] 使用 admin/admin123 登录
- [ ] 测试各项功能

## ⚠️ 常见问题解决

### 问题：部署后无法登录
**解决步骤：**
1. [ ] 检查数据库连接字符串是否正确
2. [ ] 运行数据库迁移：`npm run db:migrate`
3. [ ] 运行数据初始化：`npm run db:seed`
4. [ ] 检查JWT密钥配置

### 问题：聊天功能不工作
**解决步骤：**
1. [ ] 检查Supabase Realtime是否开启
2. [ ] 检查WebSocket连接地址
3. [ ] 检查CORS配置
4. [ ] 检查Socket.io配置

### 问题：图片无法上传
**解决步骤：**
1. [ ] 检查文件上传目录权限
2. [ ] 检查Multer配置
3. [ ] 检查存储路径配置

### 问题：性能慢
**解决步骤：**
1. [ ] 启用Vercel缓存
2. [ ] 优化数据库查询
3. [ ] 减少前端资源大小
4. [ ] 启用CDN加速
