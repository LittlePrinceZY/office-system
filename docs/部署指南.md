# 🚀 企业办公系统 - 免费部署方案（小白版）

## 📋 部署架构概览

```
用户访问 → Vercel（前端） → Railway（后端API） → Supabase（数据库）
```

### 免费服务对比

| 服务 | 用途 | 免费额度 | 优点 |
|------|------|----------|------|
| **Vercel** | 部署前端React应用 | 永远免费 | 自动部署、全球CDN、SSL证书 |
| **Railway** | 部署Node.js后端 | $5信用/月 | 自动配置、简单易用 |
| **Supabase** | PostgreSQL数据库 | 500MB存储 | 实时数据、管理界面 |
| **CloudFlare** | 域名解析（可选） | 永远免费 | 免费CDN和SSL |

## 第3步：部署前端到Vercel

### 3.1 注册Vercel

1. 访问 https://vercel.com
2. 点击 "Sign Up"
3. 选择 "Continue with GitHub"
4. 授权GitHub账号

### 3.2 创建Vercel项目

1. 点击 "Add New..." → "Project"
2. 选择你的 `office-system` 仓库
3. 配置设置：
   - **Framework Preset**: Vite
   - **Build Command**: `npm run build`
   - **Output Directory**: `apps/web/dist`
   - **Install Command**: `npm install`
4. 环境变量（重要！）：
   ```
   VITE_API_BASE_URL = https://你的后端地址.railway.app/api
   VITE_WS_URL = https://你的后端地址.railway.app
   ```
   （先创建项目，等下再填）

5. 点击 "Deploy"

### 3.3 获取部署地址
部署完成后，Vercel会给你一个地址：
`https://office-system.vercel.app`

## 第4步：部署数据库到Supabase

### 4.1 注册Supabase

1. 访问 https://supabase.com
2. 点击 "Start your project"
3. 使用GitHub登录

### 4.2 创建项目

1. 点击 "New Project"
2. 填写：
   - **Name**: `office-system-db`
   - **Database Password**: 设置一个强密码
   - **Region**: 选择离你近的地区（如 `Singapore`）
3. 点击 "Create new project"

### 4.3 获取数据库连接信息

创建完成后，点击左侧 "Settings" → "Database"
找到 "Connection string"：

```
postgresql://postgres:[密码]@[hostname].supabase.co:5432/postgres
```

### 4.4 导入数据库Schema

1. 点击左侧 "SQL Editor"
2. 新建一个查询
3. 复制 `apps/api/prisma/schema.prisma` 文件中的SQL语句
4. 执行SQL创建表

或者使用Prisma连接：

```bash
# 安装依赖
npm install

# 设置数据库URL环境变量
set DATABASE_URL=你的supabase连接字符串

# 生成Prisma客户端
npm run db:generate

# 运行迁移
npm run db:migrate
```

## 第5步：部署后端到Railway

### 5.1 注册Railway

1. 访问 https://railway.app
2. 点击 "Start a New Project"
3. 使用GitHub登录

### 5.2 创建项目

1. 点击 "New Project" → "Empty Project"
2. 点击 "GitHub" → 选择你的仓库
3. 部署配置：
   - **Root Directory**: `apps/api`
   - **Build Command**: `npm install && npm run build`
   - **Start Command**: `npm start`

### 5.3 配置环境变量

在Railway项目页面：
1. 点击 "Variables" 标签
2. 添加以下变量：

```
DATABASE_URL=你的supabase连接字符串
JWT_SECRET=生成一个随机字符串（如：office-system-2025-secret-key）
JWT_REFRESH_SECRET=另一个随机字符串
PORT=3001
NODE_ENV=production
CORS_ORIGIN=https://你的前端地址.vercel.app
```

### 5.4 部署

1. 点击 "Deploy"
2. 等待部署完成，获取后端地址：
   `https://你的项目名.up.railway.app`

## 第6步：更新前端配置

### 6.1 更新Vercel环境变量

回到Vercel项目页面：
1. 点击 "Settings" → "Environment Variables"
2. 更新：
   ```
   VITE_API_BASE_URL = https://你的后端地址.railway.app/api
   VITE_WS_URL = https://你的后端地址.railway.app
   ```
3. 点击 "Redeploy"

### 6.2 配置WebSocket（重要）

为了支持实时聊天，需要配置Supabase的Realtime：

1. 在Supabase控制台：
   - 点击左侧 "Database" → "Replication"
   - 开启 "Realtime" 开关
   - 配置要监听的表：
     - `notifications`
     - `messages`
     - `announcements`

2. 更新后端代码中的Socket.io配置：
   ```javascript
   // 允许Vercel和Railway的跨域
   const io = new Server(server, {
     cors: {
       origin: [
         "https://你的前端.vercel.app",
         "https://你的后端.railway.app"
       ],
       credentials: true
     }
   });
   ```

## 第8步：自定义域名（可选）

### 8.1 购买域名

推荐在阿里云或腾讯云购买：
- 价格：约10-50元/年
- 建议域名：`你的公司名-tech.cn` 或 `oa-你的名字.com`

### 8.2 配置Vercel域名

1. 在Vercel项目页面：
   - 点击 "Settings" → "Domains"
   - 输入你的域名
2. 在域名注册商处配置DNS：
   - 添加CNAME记录：
     ```
     主机名：@
     类型：CNAME
     值：cname.vercel-dns.com
     ```

### 8.3 配置Railway域名

1. 在Railway项目页面：
   - 点击 "Settings" → "Custom Domains"
   - 添加子域名：`api.你的域名.com`
2. 配置DNS：
   - 添加CNAME记录：
     ```
     主机名：api
     类型：CNAME
     值：你的项目名.up.railway.app
     ```

## 第10步：常见问题解决

### ❌ 问题1：部署后无法登录
**原因**：数据库未初始化
**解决**：
```bash
# 连接到Supabase数据库
npm run db:migrate
npm run db:seed
```

### ❌ 问题2：聊天功能不工作
**原因**：WebSocket配置错误
**解决**：
1. 检查Supabase Realtime是否开启
2. 检查Railway环境变量CORS_ORIGIN是否正确
3. 检查前端WebSocket连接地址

### ❌ 问题3：图片上传失败
**原因**：存储路径权限问题
**解决**：
1. Railway项目添加持久化存储卷
2. 或配置Supabase Storage存储图片

### ❌ 问题4：性能慢
**原因**：免费套餐限制
**解决**：
1. 启用Vercel和Railway的缓存
2. 减少前端打包体积
3. 优化数据库查询
